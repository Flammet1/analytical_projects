Разработка A/B-тестирования и анализ результатов
Описание проекта
В данном проекте мы исследуем результаты A/B-тестирования нового алгоритма рекомендаций в развлекательном приложении с функцией «бесконечной» ленты. Основная цель исследования — оценить эффективность нового алгоритма и его влияние на поведение пользователей.

Набор данных
Используются три таблицы:

sessions_project_history.csv — исторические данные по сессиям пользователей за период с 2025-08-15 по 2025-09-23.

sessions_project_test_part.csv — данные за первый день проведения A/B-теста (2025-10-14).

sessions_project_test.csv — данные за весь период проведения A/B-теста (с 2025-10-14 по 2025-11-02).

Поля таблиц включают:

user_id — идентификатор пользователя.

session_id — идентификатор сессии.

session_date — дата сессии.

session_start_ts — дата и время начала сессии.

install_date — дата установки приложения.

session_number — порядковый номер сессии.

registration_flag — флаг регистрации пользователя.

page_counter — количество просмотренных страниц.

region — регион пользователя.

device — тип устройства.

test_group — тестовая группа (только в тестовых данных).

Основные этапы исследования
Загрузка и предварительный анализ данных:

Импорт данных и знакомство со структурой.

Проверка уникальности пользователей и сессий.

Анализ количества регистраций пользователей.

Визуализация данных:

Построение графиков для анализа динамики регистраций и активности пользователей.

Сравнение поведения пользователей в тестовой и контрольной группах.

A/B-тестирование:

Определение ключевых метрик (например, количество просмотренных страниц, активность пользователей).

Проверка статистической значимости различий между группами.

Используемые библиотеки
pandas — для работы с данными.

matplotlib и seaborn — для визуализации.

numpy — для численных операций.

scipy и statsmodels — для статистического анализа.

Выводы
Исторические данные:

Пользователи с наибольшим количеством сессий чаще используют приложение без регистрации.

Динамика регистраций показывает устойчивый рост, но доля зарегистрированных пользователей остается относительно низкой.

A/B-тестирование:

Новый алгоритм рекомендаций показал значительное влияние на поведение пользователей. Например, увеличение среднего количества просмотренных страниц на X% в тестовой группе по сравнению с контрольной.

Статистический анализ подтвердил значимость различий (p-value < 0.05), что позволяет сделать вывод об эффективности нового алгоритма.

Рекомендации:

Внедрить новый алгоритм рекомендаций для всех пользователей, так как он улучшает ключевые метрики.

Провести дополнительное исследование для оптимизации алгоритма под разные группы пользователей (например, по регионам или типам устройств).

Исследование подтвердило гипотезу о том, что новый алгоритм рекомендаций повышает вовлеченность пользователей, что может положительно сказаться на монетизации приложения.
